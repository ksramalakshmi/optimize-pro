CSV Upload Guidelines & Documentation

Format Overview

Your CSV file must include a header row with specific column names. The system uses these headers to map your data correctly.

Required Columns

  • sku: A unique identifier for the product (e.g., WH-1000).
  • name: The full name of the product.
  • cost_price: Your base cost to acquire or manufacture the unit. Must be a positive number.
  • quantity: The total starting stock value available in your warehouse.

Optional Columns

  • category: A generic categorization tag (e.g., Electronics, Apparel).
  • [marketplace_code]_price: To automatically set initial pricing on your marketplaces, you can append `_price` to the marketplace code you've set up. E.g., if you have an `amazon` code, using `amazon_price` will set the listing price for that marketplace during upload.

Do's and Don'ts

Do's ✅
  • Always verify your CSV headers exactly match the required/optional names (case-sensitive, lowercase).
  • Ensure numeric columns (cost_price, quantity) contain only valid numbers, decimals are allowed for price.
  • Save your file explicitly as `.csv` format encoded in UTF-8.
Don'ts ❌
  • Don't include currency symbols like ₹, $, or commas in your numerical data columns.
  • Don't leave required fields blank—the entire row will fail to import if SKU, Name, Cost Price, or Quantity are missing.
  • Don't include duplicate SKUs within the same CSV file to avoid conflicts.
